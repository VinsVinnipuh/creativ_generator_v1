# Telegram бот для генерації Instagram креативів

Цей проєкт — готова основа Telegram-бота на Python, який:

1. Приймає від користувача бриф (публічну інформацію про продукт/послугу).
2. Приймає візуальні референси (фото).
3. Генерує кілька варіантів креативів для Instagram за допомогою OpenAI API.

## Можливості

- Команди:
  - `/start` — коротка інструкція.
  - `/new` — очищення поточного брифу та референсів.
  - `/generate` — генерація креативів на основі зібраного контексту.
- Підтримка мультимодального вводу:
  - Текстові повідомлення додаються до брифу.
  - Фото в Telegram додаються як референси (через file URL).
- Налаштування кількості варіантів через змінну `CREATIVE_VARIANTS`.

## Швидкий старт

### 1) Підготовка

```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

### 2) Налаштуй змінні середовища

Створи `.env` файл:

```env
TELEGRAM_BOT_TOKEN=your_telegram_bot_token
OPENAI_API_KEY=your_openai_api_key
OPENAI_MODEL=gpt-4.1-mini
CREATIVE_VARIANTS=3
```

> `OPENAI_MODEL` можна змінити на іншу мультимодальну модель OpenAI.

### 3) Запуск

```bash
python -m bot.main
```

### 4) Деплой у Railpack

Щоб Railpack коректно запустив сервіс, у репозиторій додано `railpack.json` зі `startCommand` (`python -m bot.main`).
Також додано root-`main.py` як fallback entrypoint.


## Приклад сценарію використання

1. Надішли боту текст з описом продукту, ЦА, оферу.
2. Надішли 1–5 фото-референсів.
3. Введи `/generate`.
4. Отримай кілька готових креативних концепцій у структурованому форматі.

## Важливо

- Передавай **лише публічну або дозволену до обробки інформацію**.
- Зберігання стану зроблене в пам'яті процесу (`in-memory`). Для продакшену краще додати БД (Redis/PostgreSQL).
